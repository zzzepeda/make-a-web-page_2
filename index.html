Intro to Programming Notes

HTML Basics and the Web

The web is a collection of HTML documents. HTML stands for Hyper Text Markup Language and makes up basically every page on the web, approximately 30 billion webpages. Hyperlinks between pages makes the web, weblike.

To access the web, your computer's browser makes requests via the Internet to Servers, which are computers designed to host files, using a protocol called HTTP. Servers respond with files that your browser displays as web pages. HTML is made up of:

  • Text Content which you can see
  • Markup which is the HTML language
    • It is important to note that "computer's are stupid". That is computer's are only able to interpret instructions literally.
  • References to other images and videos
  • Links to other pages

HTML Tags, Elements, and Attributes

HTML Markup is made up of things called Tags. An HTML tag is always contained within angled brackets. Most tags have an opening tag <p> for example and a closing tag, </p>. Some tags (called "void" tags) do not require a closing tag, like the <br> tag. Tags can either be Inline or Block. Inline tags only modify whatever is in the same line as it. Block tags however create an invisible box and affect whatever is contained in it. More on this later.

An Element refers to everything within a set of opening and closing tags.

Memorizing all the tags there are in HTML is impossible so it is important to remember to utilize google when you've forgotten the details.

Attributes are a proprty of an HTML element. For example, to set the attribute href to the anchor tag with the Udacity URL, you would write <a href="www.udacity.com">

HTML Structure and DOM

HTML is the language of the web and it has syntax and rules. The structure of HTML is tree-like. The browser uses HTML to create a Document Object Model (DOM). In Developer's Tools you can see the DOM the browser built from HTML. Each HTML tag creates an element in the DOM that the browser uses to display the page.

The basic structure of an HTML document is as follows:

<!DOCTYPE html> /*this tells the browser this is an HTML document
<html>
<head>
/*this is where the link to the style file will be located. Text does not show up on browser
<title>
</head>
<body>
/*This is where content will go
</body>
</html>

Using Semantic Tags Make sure to choose the right type of element for certain parts of a page depending on what the purpose of that element is. e.g. the heading tag used for the heading of the page. Search engines use headings to index the structure and contents of a web page.

Boxes

Everything on a webpage is contained in a box. Div tags define boxes and can have attributes. More on this later. Just be aware that everything on a web page is made up of boxes.

CSS Styling and HTML Structure

Cascading Style Sheets (CSS) is the language that controls the style of HTML. CSS can either be put directly into HTML code or it can have its' own style sheet. Generally, having a seperate style sheet is preferable to prevent repetition (Inheritance is an important concept in preventing repetition. More information can be found here.) The link to the style sheet is placed in the <head> tag (see above). The code to link the style sheet is:

<link rel="stylesheet" type="text/css" href="filename.css">

Selectors

The biggest key to understanding CSS is understanding selectors. Selectors are what allow you to target specific HTML elements and apply style to them. For example:

h1 {
background-color: black;
}

In this example, everything in the h1 element would be affected. “background-color : black;” is a declaration “background-color” is a property and “black” is the value of that property. Here is a link to various attributes that include font style, colors, etc. CSS Attribute Reference

Selecting by Class

Classes are used to style a group of elements similarly. Let's use the analogy of a floorboard in a house. Each floorboard can represent an element and we want to assign all of these elements class="floorboard" where the floorboard styling rule will contain the color to be "brown." If we want to style a special floorboard, such as a floorboard that creaks, we can use another class called "creak" and add in that additional style such as: <div class="floorboard creaks">

In the prior example the background of EVERY h1 will be black. If we want to add style to a special h1, it will be necessary to use Class Selectors. First set the HTML element to a class selector to match the CSS code.
For example:

<div class=“h1-special”>text</div>

The CSS code will then be:

.h1-special {
attribute : value
}

Using the appropriate selector, you can apply rules to ALL div’s or just certain div’s, as in the above case. Think of

Div, Span, ID & Class

  • Div is used to logically organize code, to group items together so styling to the whole block of code at once and avoid repetition.
  • Span is used to style parts of the text or a few elements together w/out affecting the layout of the page.
  • Ids are used to identify a unique element in the entire page. It should only be used sparingly because it slows down the browser. With ids, anchor elements can be used to directly link the unique ID such as: <div id="creaks" class="floorboard">
    • It's recommended to use classes as much as possible in our HTML to style our elements because it's efficient for browsers to style elements belonging to the same class and it makes our code easier to understand and maintain.

The Box Model

In Dev Tools, Computing tab, you’ll find a box.

Content is what shows up on the webpage, image or text. Think of the Content as the Nucleus and the other elements protect it and make the page appealing. Padding clears an area and protects the contents; is affected by the background color of the box. Border goes around padding and content. Border is inherited from the color property of the box. Margin clears the area around the border and doesn’t have a background color. Space between boxes. By default the size of your element (box) is equal to the sum of the border, padding and contents widths.

In order to not have to re-calculate each time you change the value, use CSS styling:
{
box-sizing: border-box;
}

This makes the calculation of the element size include both the border and padding. You can just set the overall width of the box and no matter how you tweak the padding and border sizes the size of the box will always stay the same. Margin is not included however. So for older browsers you must add “browser specific prefixes. Add before the style definition.
{
-webkit-box-sixing: border-box;
-moz-box-sizing: border
-ms-box-sizing: border-box;
box-sizing: border-box
}

Box Sizing: Pixels vs. Percentages

Boxes can be defined in pixels or percentages. If you set the box to percentages the size will change depending on how big the screen is or how encompassing the box is. If you set as a predefined # of pixels it will stay exactly the same and not change no matter how big or small the screen is - can lead to bad user experience.

You can also set a max width for the box. In that case it will resize and become smaller than necessary but not larger than the set max width. For example: .screenshot {
max-width:460px
}

4 Main Points:

  • HTML elements are boxes and each box has 4 components.
  • Because there are so many components to each box, it can often be hard to get the size of a box just right.
  • There are two techniques you can use to help deal with sizing issues.
    • Set sizes in terms of percentages rather than pixels.
    • Set the box-sizing attribute to border-box for every element.
  • Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.

Box Positioning

Divs are block elements (as opposed to inline), so by default they take up the entire width of a page. Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other. For example:

.app {
display: flex;
}

Code, Test, Refine

  1. Look for natural boxes.
  2. Look for repeated styles and semantic elements (h2’s should be the 2nd biggest heading etc.)
  3. Write your HTML
  4. Apply Style from biggest to smallest
  5. Fix things (look in other browsers, repeat 3-5 until it looks as you like)
  6. You can also use the Dev Tools to test/change the page without having to change the HTML or CSS code. You just have to refresh the page to get back to the original code. You can then experiment until you get your desired result and then go back and change the HTML or CSS.
